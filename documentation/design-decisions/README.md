# Design Decisions

This is mostly human written. Poorly written. 

Typically brain to paper decisions.

These documents are things and thoughts and decisions that typically aren't directly discoverable in code. This is context that is valuable to agentic coding harnesses, the rest they can figure out from code.

Design decisions rarely change, much more static and easier to keep up to date than exact docs on how everything works.

Curious how to use a feature? Curious how something works? Ask an agent to read the code and tell you.
